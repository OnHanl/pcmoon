# PC Master — сайт + полноценная CRM-панель для сборки и ремонта компьютеров

Django-проект: публичный сайт с прайсом, портфолио, отзывами и формой заявки,
плюс отдельная тёмная CRM-панель (`/dashboard/`) — там ведётся вся повседневная
работа: заявки на канбан-доске, прайс-лист, портфолио, отзывы, FAQ, контакты.
Заходить в стандартную Django-админку (`/admin/`) для повседневных задач не
нужно — она оставлена как технический fallback (управление пользователями и т.д.).

## Что внутри

### Публичный сайт (`orders/templates/orders/index.html`)
- Hero с мгновенным калькулятором цены (устройство + проблема → вилка цен)
- Прайс-лист по категориям (вкладки)
- Примеры работ со слайдером «до / после» (фото «до» — по желанию)
- Отзывы клиентов с рейтингом и меткой услуги
- FAQ-аккордеон
- Форма заявки (имя, телефон, услуга, комментарий)
- Блок контактов, плавающие кнопки связи

### CRM-панель (`/dashboard/`, только для staff-пользователей)
Единая тёмная панель, все разделы в одном стиле — не нужно уходить в
стандартную Django-админку:

- **Заявки (Kanban)** — `/dashboard/`
  Карточки заявок перетаскиваются мышкой между статусами (Новая → Нужно
  перезвонить → Подтверждена → В работе → Ожидание запчастей → Готово →
  Завершена), статус реально сохраняется в базе при каждом перетаскивании.
  Клик по карточке открывает детали + журнал звонков — можно добавить запись
  о звонке прямо там. Карточки со статистикой сверху (новые заявки за
  сегодня, в работе, выручка за месяц, активных всего) считаются от реальных
  данных.
- **Прайс-лист** — `/dashboard/prices/`
  Категории и услуги, добавление/редактирование/удаление прямо в панели.
- **Примеры работ** — `/dashboard/portfolio/`
  Загрузка фото «до» и «после» (для слайдера сравнения на сайте), метки,
  публикация/скрытие.
- **Отзывы** — `/dashboard/reviews/`
  Имя клиента, текст, оценка, метка услуги, флаг «подтверждённый заказ».
- **FAQ** — `/dashboard/faq/`
  Вопросы и ответы для блока на сайте.
- **Контакты** — `/dashboard/contact/`
  Единая форма редактирования телефона, telegram, whatsapp, адреса, часов
  работы.

### Стандартная админка (`/admin/`)
Оставлена как технический резерв — управление пользователями/правами,
таблица заявок с фильтрами и массовыми действиями, журнал звонков отдельным
списком. Из списка заявок есть кнопка «Открыть Kanban-панель» — переход в
основной рабочий интерфейс.

## Запуск локально

```bash
python3 -m venv venv
source venv/bin/activate        # Windows: venv\Scripts\activate
pip install -r requirements.txt

python3 manage.py migrate
python3 manage.py createsuperuser   # создать логин/пароль для входа

python3 manage.py runserver
```

Сайт: http://127.0.0.1:8000/
CRM-панель: http://127.0.0.1:8000/dashboard/ (нужен вход — используйте
логин/пароль суперпользователя, форма входа стандартная Django)
Служебная админка: http://127.0.0.1:8000/admin/

## Демо-доступ (в этой сборке)

В базе уже есть демо-данные (прайс, примеры работ, отзывы, заявки на разных
стадиях) и суперпользователь:

- **Логин:** `admin`
- **Пароль:** `admin12345`

Обязательно смените пароль или создайте нового суперпользователя перед
реальным использованием (`python3 manage.py changepassword admin`).

## Как наполнять сайт

Всё содержимое редактируется через CRM-панель, без правки кода:

1. Зайдите в `/dashboard/` (нужен логин staff-пользователя)
2. **Прайс-лист** → создайте категории («Сборка ПК», «Ремонт» и т.д.),
   затем добавьте услуги с ценами внутри них
3. **Примеры работ** → загрузите фото (можно и «до», и «после» — тогда на
   сайте появится слайдер сравнения), опишите работу и добавьте метки
4. **Отзывы** → добавляйте отзывы клиентов вручную
5. **FAQ** → заполните частые вопросы
6. **Контакты** → укажите телефон, telegram, часы работы

## Как работать с заявками

1. Клиент отправляет форму на сайте → заявка появляется на канбан-доске в
   колонке «Новая заявка»
2. Открываете карточку, звоните клиенту
3. Добавляете запись в журнал звонков прямо в открытой карточке
4. Перетаскиваете карточку в нужный статус («Подтверждена» → «В работе» → …)
5. По завершении — в колонку «Завершена», при необходимости указываете
   итоговую стоимость через стандартную админку (`/admin/orders/order/`),
   там же можно выставить приоритет заявки

## Продакшн

Перед выкладкой в открытый доступ обязательно:

- Установите `DEBUG = False` в `pcservice/settings.py`
- Задайте `ALLOWED_HOSTS`
- Вынесите `SECRET_KEY` в переменную окружения
- Настройте отправку статики/медиа (whitenoise, S3 или через nginx)
- Подключите нормальную БД (Postgres) вместо sqlite3, если нагрузка
  предполагается серьёзная — для одного мастера sqlite тоже вполне подойдёт

## Структура проекта

```
pcservice/              # настройки проекта Django
orders/                 # основное приложение
  models.py             # Заявка, Услуга, Портфолио, Отзыв, FAQ, Контакты, ЖурналЗвонков
  admin.py               # стандартная Django-админка (технический резерв)
  views.py               # публичный сайт + вся CRM-панель (kanban + CRUD)
  forms.py                # форма заявки + формы CRM-панели
  urls.py                  # маршруты сайта и /dashboard/...
  templates/orders/
    base.html, index.html    # публичный сайт
    crm_base.html             # общая тёмная оболочка CRM (сайдбар/топбар)
    dashboard.html             # kanban-доска заявок
    crm_prices.html            # список прайса
    crm_portfolio.html         # список примеров работ
    crm_reviews.html           # список отзывов
    crm_faq.html                # список FAQ
    crm_form.html                # универсальная форма добавления/редактирования
static/css/style.css    # стили публичного сайта
static/js/main.js       # интерактив публичного сайта (меню, табы, FAQ, слайдер)
media/                   # загруженные фото (примеры работ)
```
